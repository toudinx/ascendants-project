# Diretrizes do Jogo (SoT de filosofia + escopo)

**Authority:** Source of Truth (pilares, guardrails e não-metas)  
**Status:** Ativo  
**Escopo:** O que o jogo é/não é. Ritmo, foco, limites de escopo.  
**Projeto:** KAEZAN: AWAKENING  
**Última revisão:** 2025-12-17


## 1) DNA do MVP
- Auto-battle **tick-based**
- Runs **curtas**
- Buildcraft **leve** (decisões claras, sem árvore gigante)
- Foco em **clareza** e **feedback** (UI + combate legível)

## 2) Restrições (para evitar expansão acidental)
- Não criar novos modos/sistemas “paralelos” no MVP.
- Nada de capítulos/campanha longa (run é a unidade).
- Qualquer expansão deve respeitar:
  - mesma régua numérica (ver números)
  - mesmo vocabulário canônico (ver canon)

## 3) Loop macro (MVP)
- Home → seleção de modo → seleção de Kaelis → setup → run → recompensa → home

## 4) Ritmo do combate
- Tick = unidade de feedback/step (tuning).
- Turno = unidade lógica de ação (cooldowns e buffs são em **turnos**).

## 5) Anexo — Gameplay Base (apoio)
(derivado do doc antigo “GAMEPLAY BASE”)

### Premissas
- Tick-based “explosivo”: micro-resoluções frequentes para manter presença na tela.
- Buildcraft leve: poucas escolhas por sala, sempre com impacto fácil de ler.

### Tick (tuning)
- Exemplo antigo citava 0.75s; tratar como **tuning**.
- O doc de combate/números usa “turno” como unidade lógica.

### O que precisa ficar sempre consistente
- Logs e efeitos devem se alinhar ao mesmo modelo: **Turno** (lógica) e **Tick** (feedback).
